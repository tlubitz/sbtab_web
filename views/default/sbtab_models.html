    SBtab Models

    SBtab files describing biochemical network models

    For the conversion of an SBtab file or several SBtab files, the provision of a 'Reaction' SBtab is crucial, since it defines the strucure of the new SBML model. But other SBtabs can be included in the conversion as well (e.g. TableType='Compound' or TableType='Compartment'). Please note here that it is crucial to stick to the identifying names: If the sum formula of a reaction lists 'ATP' as a reactant, the corresponding species in the Compound SBtab needs to be named 'ATP' as well (and not for instance 'Adenosine-triphosphate').

    Here is an example:

    !!SBtab TableType="Reaction" SBtabVersion="0.9.1"
    !ID !ReactionFormula !Identifiers:kegg.reaction !GeneName
    R1 ATP + F6P <=> ADP + F16P R00658 pfk
    R2 F16P + H2O <=> F6P + Pi R01015 fbp
    !!SBtab TableType="Compound" SBtabVersion="0.9.1"
    !ID !Name !InitialConcentration !Location
    F6P Fructose-6-phosphate 0.1 cytosol
    ATP ATP 0.2 cytosol
    ADP ADP 1.1 cytosol
    F16P Fructose-1,6-bisphosphate 1.2 cytosol
    H2O Water 10.0 cytosol
    Pi Inorganic phosphate 2.1 cytosol
    PEP Phosphoenolpyruvate 3.3 cytosol
    AMP AMP 2.1 cytosol
    !!SBtab TableType="Compartment" SBtabVersion="0.9.1"
    !ID !Size
    extracellular 2.0
    cytosol 1.0

    What does the example show?

    • The first line of an SBtab file is supposed to be the definition of the SBtab. It starts with '!!SBtab' and holds several attributes on the SBtab. (e.g. '!!SBtab SBtabVersion="0.9.1" TableType="Reaction" '). The TableType attribute must be provided, other attributes are optional.
    • The second line of an SBtab file provides the column headers of the table, each of which are preceded by an exclamation mark (e.g. '!ReactionFormula').
    • The following lines usually hold the value rows. It is absolutely crucial that they consist of as many columns as the column header line.
    • A column "!ID", if it exists, contains shortnames that serve as unique identifiers within the SBtab document.
