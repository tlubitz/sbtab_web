SBtab - Standardised data tables for Systems Biology

SBtab files describing biochemical network models

For the conversion of an SBtab file or several SBtab files, the provision of a 'Reaction' SBtab is crucial, since it defines the strucure of the new SBML model. But other SBtabs can be included in the conversion as well (e.g. TableType='Compound' or TableType='Compartment'). Please note here that it is crucial to stick to the identifying names: If the sum formula of a reaction lists 'ATP' as a reactant, the corresponding species in the Compound SBtab needs to be named 'ATP' as well (and not for instance 'Adenosine-triphosphate').

Here is an example:

!!SBtab TableType="Reaction" Version="0.1" Level="1.0"
!Reaction!SumFormula!MiriamID::urn:miriam:kegg.reaction!GeneName
R1ATP + F6P <=> ADP + F16PR00658pfk
R2F16P + H2O <=> F6P + PiR01015fbp

!!SBtab TableType="Compound" Version="0.1" Level="1.0"
!Compound!Name!InitialConcentration!Location
F6PFructose-6-phosphate0.1cytosol
ATPATP0.2cytosol
ADPADP1.1cytosol
F16PFructose-1,6-bisphosphate1.2cytosol
H2OWater10.0cytosol
PiInorganic phosphate2.1cytosol
PEPPhosphoenolpyruvate3.3cytosol
AMPAMP2.1cytosol

!!SBtab TableType="Compartment" Version="0.1" Level="1.0"
!Compartment!Size
extracellular2.0
cytosol1.0

What does the example show us?

  • The first line of an SBtab file is supposed to be the definition of the SBtab. It starts with '!!SBtab' and holds several attributes on the SBtab. (e.g. '!!SBtab Version="0.1" Level ="1.0" TableType="Reaction" '). The TableType attribute must be provided, other attributes are optional.
  • The second line of an SBtab file provides the column headers of the table, each of which are preceded by an exclamation mark (e.g. '!Reaction').
  • The following lines usually hold the value rows. It is absolutely crucial that they consist of as many columns as the column header line.
  • The first column of the SBtab table is supposed to be the leading column information in correspondance to the SBtab TableType (e.g. an SBtab of the TableType 'Reaction' should provide '!Reaction' as its first column). This first column holds the identifier for the information row and must never be empty.
